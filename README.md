# 连连看小游戏交付清单

这是一个纯前端连连看 PWA 小游戏，可直接部署到 HTTPS 静态站点，并支持手机添加到桌面。

## 功能验收清单

- 基础玩法：6 x 8 棋盘，成对图案随机生成。
- 消除规则：两个相同图案在不超过两个转角可连接时消除。
- 选择反馈：选中、取消、匹配失败、匹配成功都有即时提示。
- 路径展示：成功消除时显示连接路径动画。
- 游戏状态：展示用时和剩余方块数量。
- 重开游戏：支持一键重置棋盘和计时。
- 完成状态：全部消除后停止计时并展示完成提示。
- 移动端适配：棋盘、按钮、统计区在手机屏幕下可用。
- PWA 能力：包含 manifest、应用图标、service worker、安装入口和离线缓存。
- 离线体验：已缓存的核心资源可在无网络时打开。

## 上线前检查

- 使用本地静态服务器验证页面，而不是直接双击打开 HTML。
- 检查桌面浏览器和手机浏览器的棋盘布局是否完整。
- 检查点击、消除、重开、完成后的计时停止是否正常。
- 检查刷新页面后 service worker 是否注册成功。
- 检查 Android Chrome 是否出现“安装应用”或“添加到主屏幕”入口。
- 检查 iPhone Safari 是否可通过分享菜单添加到主屏幕。
- 检查 `manifest.webmanifest` 的名称、主题色、图标路径是否符合最终产品信息。
- 每次修改静态资源后，更新 `sw.js` 中的缓存版本号，避免用户继续使用旧版本。

## 部署方案

推荐部署到任意 HTTPS 静态网站服务：

- GitHub Pages：适合免费公开项目，上传整个目录后启用 Pages。
- Vercel：适合快速预览和正式域名绑定，项目类型选择静态站点即可。
- Netlify：适合拖拽目录部署，也支持自定义域名。
- 自有服务器：使用 Nginx/Apache 托管整个目录，并开启 HTTPS。

部署产物必须包含：

- `index.html`
- `styles.css`
- `game.js`
- `manifest.webmanifest`
- `sw.js`
- `icons/icon-192.png`
- `icons/icon-512.png`
- `手机安装说明.md`

本地验收命令：

```bash
python3 -m http.server 4173
```

然后打开：

```text
http://localhost:4173/
```

## 数据与备份方案

当前版本没有账号系统、服务端接口、数据库或用户上传内容，因此没有业务数据需要定时备份。

需要备份的是项目源文件和上线产物：

- 源码备份：把项目放入 Git 仓库，至少保留主分支和发布标签。
- 发布备份：每次上线前保留一份完整目录压缩包，文件名建议包含日期和版本号。
- 图标备份：保留 `icons` 原始设计文件或高清源图，避免后续无法重新生成。
- 配置备份：保留域名、HTTPS 证书、部署平台配置和访问权限记录。
- 回滚方案：保留最近 3 个可用版本，线上异常时直接回滚到上一版静态目录。

如果后续增加排行榜、用户成绩、关卡进度或账号功能，需要新增：

- 数据库存储方案。
- 每日自动备份。
- 备份恢复演练。
- 隐私说明和数据删除机制。

## 发布建议

- 正式名称建议去掉“AI 编程测试”字样，改为面向玩家的产品名。
- 若面向儿童或公开用户，建议增加音效开关、难度选择、重新洗牌、无解检测和隐私说明。
- 若要作为作品展示，建议补充二维码入口、版本号和简单的更新记录。
